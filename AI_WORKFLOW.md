# AI Engineering Workflow — BiztelAI DocFlow

## AI Tools Used
- Google Gemini 1.5 Flash (for OCR and data extraction)
- NVIDIA Llama 3.2 90B Vision (secondary extraction provider)
- AI Coding Agent (for autonomous coding and execution)

## Development Workflow
The project was built using a phase-by-phase master prompt approach. At the beginning of each phase, the agent read the entire codebase and the `BRAIN.md` file to establish persistent context. `BRAIN.md` served as the central source of truth for architecture, schema, endpoints, and roadmap. 
A highly detailed implementation plan was generated by the AI before any code was written, allowing for human review. Once approved, the agent autonomously executed the tasks step-by-step while maintaining a task checklist.

## Prompting Strategy
- **Phase-by-phase execution**: Complex tasks were broken down into manageable phases (e.g., scaffolding, AI integration, premium UI).
- **Persistent memory**: `BRAIN.md` acted as a living document to maintain context across different agent sessions and avoid context amnesia.
- **Strict design system enforcement**: Explicit CSS and tailwind instructions were provided within the prompt to enforce a cohesive "Glassmorphism" UI.

## Where AI Helped Most
- Gemini Vision extraction prompt engineering to parse semi-structured manufacturing documents.
- Boilerplate generation (FastAPI routers, complex Pydantic schemas).
- Recharts component setup and complex SQLAlchemy aggregation logic.
- Translating a design system into functional React components.

## Where Manual Intervention Was Needed
- Debugging SQLAlchemy join queries and fixing schema serialization errors.
- Fine-tuning the Gemini prompt for edge cases (e.g., date formats, null values).
- Setting up environment dependencies (`npm install recharts`) where execution policies or package managers occasionally failed.
- Parsing conversational JSON outputs from Llama Vision using robust substring methods.
- Debugging Vite proxy configurations for local static file serving (`/uploads`).
- Fixing schema validation crashes caused by missing fields (`file_path`) in Pydantic models.

## Key Insight
Using a persistent `BRAIN.md` file and requiring the AI agent to explicitly plan before coding drastically reduces hallucination and ensures large, multi-file refactors stay aligned with the core project architecture.
